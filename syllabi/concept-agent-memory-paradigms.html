Memory mechanisms for AI agents — a 4-paradigm taxonomy

The unified frame

Most “AI agent memory” mechanisms can be characterized by two dimensions: - Memory state — how historical information is stored. - Memory steering — how the stored state influences the backbone LLM’s reasoning at inference.

Under this frame, the existing literature falls into four broad paradigms. The taxonomy is from the δ-mem paper (Lei et al., 2026), with origins in earlier surveys.

The four paradigms

1. Textual Memory Mechanisms (TMM)

  • Storage: natural-language text — message logs, extracted facts, summaries.
  • Steering: re-injected into the input prompt (sometimes after retrieval).
  • Examples: Memgpt (Packer et al., 2023) (MemGPT), Generative Agents (Park et al., 2023) (Generative Agents), Mem0 (Chhikara et al., 2025) (Mem0), MemoryBank, RAG, RETRO.
  • Strengths: flexible, interpretable, requires no architectural change.
  • Weaknesses: context-window limits, retrieval noise, compaction loss (the LLM “summarizes” away information that turns out to matter later).

2. Outside-Channel Memory Mechanisms (OMM)

  • Storage: external module — a kNN cache of past key-value pairs, an autoencoded latent bank, a side network.
  • Steering: retrieved & fused via an auxiliary pathway (a separate attention head, a gated residual).
  • Examples: Memorizing Transformers (Wu et al., 2022) (Memorizing Transformers), LongMem, MLP Memory.
  • Strengths: avoids tokenization loss; the stored representation is latent and high-dimensional.
  • Weaknesses: retrieval overhead, integration complexity, possible representational mismatch between stored and current activations.

3. Parametric Memory Mechanisms (PMM)

  • Storage: in additional trainable parameters — LoRA adapters, prefix tokens, edited weights (ROME, MEMIT).
  • Steering: the parameters are part of the forward pass; the LLM is permanently modified.
  • Examples: LoRA, Prefix-Tuning, ROME, MEMIT, MemGen.
  • Strengths: zero runtime overhead; compatible with frozen backbones.
  • Weaknesses: static after training; can’t evolve with a conversation. Suited to knowledge editing, not online memory.

4. Online-State / Associative Memory Mechanisms

  • Storage: a compact, dynamically-updated state (a small matrix or an MLP) maintained alongside the backbone, updated by online learning (delta rule, gradient descent on a “surprise” loss).
  • Steering: read into the forward pass as a low-rank correction to attention components.
  • Examples: Titans (Behrouz et al., 2024) (Titans), Delta MEM (Lei et al., 2026) (δ-mem). Linear-attention models that close on the same kernel: DeltaNet, GLA, Mamba2-style.
  • Strengths: compact (a 64-scalar state is enough in δ-mem!), backbone-aligned, no retrieval overhead.
  • Weaknesses: requires training the read/write projections; throughput cost per token; explainability is poor compared to TMM.

        ┌───────┐                  │
LLM ────│ attn  │── output       ──┼─ OMM: side pathway + gate ──────────┤
        └───────┘                  │
                                  └─ Online: low-rank correction (δ-mem)─┘

                  PMM: weights themselves are the memory (LoRA / ROME)

When to use which

  • You have a clear set of long-lived facts and want explainability → TMM (Mem0, MemGPT).
  • You want maximum recall fidelity over very long histories and can afford a kNN index → OMM (Memorizing Transformers, LongMem).
  • You want to permanently teach the model new knowledge → PMM (ROME / MEMIT).
  • You want a tiny, dynamic, backbone-aligned memory state for online learning at inference time → Online-state (Titans, δ-mem).

In practice, production agents combine paradigms: TMM for user-facing facts, RAG for documents, and increasingly online-state for low-level steering.
