Associative memory and the delta rule

The intuition

An associative memory is a function that maps keys to values, where both are continuous vectors. Given a key $k$, the memory returns its associated value $v$. The classical question — going back to Hopfield, Kanerva, Anderson — is: can we store many $(k, v)$ pairs in a fixed-size state and recover them on query?

The simplest construction: store the sum of outer products.

$$S = \sum_i v_i k_i^\top, \qquad \hat{v}(k) = S k$$

If the keys are orthonormal, this is a perfect lookup table. If they are merely near-orthogonal (random vectors in high dimension), this is a noisy but useful associative recall — the foundation of Hopfield networks and the modern “linear attention as fast weights” view (Schlag, Irie, Schmidhuber 2021).

The delta rule

Plain accumulation $S \leftarrow S + v_t k_t^\top$ has two problems: it can’t correct a wrong association, and it has no notion of forgetting. The delta rule (Widrow-Hoff, 1960; the same algorithm as online LMS) fixes the first:

$$S_t = S_{t-1} + \beta_t \cdot \underbrace{(v_t - S_{t-1} k_t)}_{\text{residual}} k_t^\top$$

This is exactly SGD on the squared-error loss $\mathcal{L}_t(S) = \tfrac{1}{2}\|S k_t - v_t\|^2$, with learning rate $\beta_t$. Three properties matter:

  1. Self-correcting: if $S_{t-1} k_t$ already equals $v_t$, the update is zero. Only new information moves $S$.
  2. Capacity ≈ rank: with $k_t \in \mathbb{R}^r$, the state $S \in \mathbb{R}^{r \times r}$ has $r$-dimensional column space. You can store ~$r$ near-orthogonal associations cleanly.
  3. Local in time: each update depends only on the current $(k, v, \beta)$ and the previous state. Streaming-friendly.

Adding a forget gate

To handle drift in long sequences, add a per-step decay:

$$S_t = \lambda_t S_{t-1} + \beta_t (v_t - S_{t-1} k_t) k_t^\top$$

With $\lambda_t \in [0,1)$ this gives graded forgetting — old associations decay exponentially unless re-written. In δ-mem, both gates are dimension-wise (vectors, not scalars): some memory dimensions retain old state, others overwrite. Generalizes naturally to the gated linear-attention family (RetNet, GLA, RWKV-7, DeltaNet).

Worked micro-example

Pseudocode for a single-head online associative memory, dimensions $r=4$:

import torch

r = 4
S = torch.zeros(r, r)        # the state

def step(x, W_k, W_v, W_q, W_beta):
    k = torch.tanh(W_k @ x); k = k / k.norm()
    v = W_v @ x
    q = torch.tanh(W_q @ x); q = q / q.norm()
    beta  = torch.sigmoid(W_beta @ x)   # write gate, per dimension
    lam   = 1.0 - beta                   # forget gate, per dimension

    # 1. READ — query the previous state
    r_t = S @ q                          # ∈ R^r

    # 2. WRITE — gated delta-rule update
    residual = v - S @ k                 # ∈ R^r
    update = torch.outer(beta * residual, k)
    decay  = torch.diag(lam) @ S
    nonlocal_S = decay + update

    return r_t, nonlocal_S

This is the kernel of δ-mem (and structurally also of Titans’ MLP memory, of DeltaNet, of RetNet’s gated retention). The wrapping differs: how the read vector $r_t$ is used to influence the backbone, how the projections $W_k, W_v, W_q, W_\beta$ are trained.

How it shows up in the syllabus

  • Memorizing Transformers (Wu et al., 2022) — doesn’t use this kernel. Its memory is a non-differentiable kNN cache; this kernel was the answer to “how do we make it learnable?”.
  • Titans (Behrouz et al., 2024) — generalizes the kernel: the state is a small MLP, the update is SGD-with-momentum on a surprise loss, not just delta-rule.
  • Delta MEM (Lei et al., 2026) — uses the kernel exactly (gated delta rule), with a tiny $8\times 8$ matrix, applied as a side-state to a frozen Transformer.

Why this concept is the heart of the syllabus

The textual-memory line (MemGPT, Generative Agents, Mem0) treats memory as a database problem: insert, retrieve, summarize. The online-state line treats memory as an optimization problem: at every step, do one step of SGD on the running key→value map. The delta rule is the simplest learning rule that gets you a self-correcting, dynamically-decaying memory — which is why every modern test-time-learned memory module is built around it.
