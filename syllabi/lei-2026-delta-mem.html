Delta MEM (Lei et al., 2026)

researchSource PDF: lei-2026-delta-mem.pdf
Part of
Syllabus — Memory for AI agents

Summary

δ-mem (Lei et al., 2026) augments a frozen full-attention LLM with a compact online state of associative memory updated by a gated delta rule. Unlike textual memory (MemGPT, Mem0), outside-channel memory (Memorizing Transformers, LongMem), or static parametric memory (LoRA, ROME), δ-mem keeps memory as a tiny matrix $S_t \in \mathbb{R}^{r \times r}$ that is read into the forward pass as a low-rank correction to the query and output of attention. With $r=8$ (an $8\times 8$ matrix) the method improves the Qwen3-4B-Instruct backbone average from 46.79% to 51.66% across IFEval / HotpotQA / GPQA-D / MemoryAgentBench / LoCoMo, and 1.31× on MemoryAgentBench. Striking corollary: in a no-context setting, the $8\times 8$ state alone recovers ~6.4 EM on HotpotQA from 0.08 EM — memory truly compresses history without storing tokens. Trainable parameter overhead is 0.12% (4.87M for the $r=8$ variant on Qwen3-4B).

Key points

The unified 4-paradigm taxonomy (Section 1)

The paper’s clearest contribution may be the framing: any memory mechanism is characterized by memory state (how history is stored) and memory steering (how the state influences the backbone). Existing methods fall into three paradigms, all with named limitations:

Paradigm How memory is stored How it steers backbone Limitation
Textual (TMM) — MemGPT, Mem0, MemoryBank, RAG, RETRO as text entries / summaries re-inject into input context context-window limits, retrieval noise, compaction loss
Outside-channel (OMM) — Memorizing Transformers, LongMem, MLP Memory external module (kNN bank, side network) retrieve & fuse via auxiliary pathway overhead, integration complexity, possible representational mismatch
Parametric (PMM) — LoRA, Prefix-Tuning, ROME, MEMIT in additional weights/prefixes static modification of backbone doesn’t evolve with the sequence

δ-mem proposes a fourth: a compact online state directly coupled with attention computation.

Delta-rule online memory update

The memory state $S_t$ stores key→value associations. At each step, prediction is $\hat{v}_t = S_{t-1} k_t$. The state is updated by minimizing $\frac{1}{2}\|S k_t - v_t\|^2$ via SGD, giving:

$$S_t = \lambda_t S_{t-1} + \beta_t (v_t - S_{t-1} k_t) k_t^\top$$

with dimension-wise gates $\lambda_t, \beta_t \in \mathbb{R}^r$ generated from the hidden state via a sigmoid. The forget gate is inspired by Qwen-Next’s gated retention. Crucially, the update writes only the residual — well-learned associations induce negligible updates. This is the classical fast-weight / online associative memory recipe, applied as a side-state to a frozen Transformer.

Read = low-rank attention correction

Reading is just a matrix-vector product: $r_t = S_{t-1} q_t^m \in \mathbb{R}^r$. The read vector then drives two affine projections into the backbone: - Query correction: $\tilde{q}_t = q_t^0 + \frac{\alpha}{r} W^\Delta_q r_t$ - Output correction: $\tilde{y}_t = a_t + \frac{\alpha}{r} W^\Delta_o r_t$

The projections $W^\Delta_q, W^\Delta_o$ are fixed after training, but their input $r_t$ comes from the dynamic state, so the effective correction is history-conditioned — unlike LoRA where the update is static. Head ablation: qo is the sweet spot (47.97 avg); full qkvo only marginally better (48.05) at extra parameter cost.

Three writing granularities (Section 3.5)

  • Token-State Write (TSW) — write at every token (finest, noisiest).
  • Sequence-State Write (SSW) — average hidden states within a message segment, write once per message.
  • Multi-State Write (MSW) — $N$ parallel sub-states $\{S^{(1)}, \ldots, S^{(N)}\}$ ($N=4$ in experiments), concatenate readouts.

Empirically: TSW best on Qwen3-4B (51.66), SSW best on Qwen3-8B (50.86), MSW best on SmolLM3-3B (36.96). Capacity-matched: stronger backbones tolerate finer granularity; weaker backbones benefit from segregation.

Key experimental claims

  • Memory benchmarks (Qwen3-4B-Instruct backbone):
    • MemoryAgentBench avg: 29.54 → 38.85 (1.31×, MSW best)
    • LoCoMo avg: 40.79 → 49.12 (1.20×, MSW best)
    • TTL subtask: 26.14 → 50.50 (nearly 2×, SSW)
  • General capability preserved: IFEval 81.89 → 82.99 (TSW), GPQA-D 39.39 → 41.41 (SSW).
  • Beats all four baseline families on the avg metric: BM25 RAG, LLMLingua-2, MemoryBank (textual); Context2LoRA, MemGen (parametric); MLP Memory (outside-channel).
  • No-context recovery (Section 5.1, the most striking result): with original context stripped, the 8×8 state lifts HotpotQA EM from 0.08 → 6.48 and F1 from 8.27 → 15.20. State really stores reusable historical signal.
  • Insertion depth ablation: all-layers > middle-12 > front-12 > back-12. Middle layers are the most effective single locus.
  • Parameter overhead: 4.87M params (0.12% of 4B backbone) for SSW/TSW; 19.47M for MSW. Compare MLP Memory at 3,078M (76.4% of backbone).
  • Inference efficiency: GPU memory matches Vanilla / Context2LoRA even at 32k prompts; decode TPS lower than Vanilla (must read/update state per step) but well above MemGen.

Limitations and honest gaps

  • Training is on QASPER (2,219 samples, 8k context) — relatively small SFT regime; transfer to longer agent trajectories is asserted, not proven at scale.
  • No comparison against Titans (Behrouz et al. 2024) — the closest neural-memory predecessor — even though both share the test-time learning premise.
  • Throughput cost of the per-token state update is real (TPS drops vs. Vanilla); not benchmarked at high concurrency or with batched serving.
  • The “1.31× / 1.20×” gains are on benchmarks the authors picked; LongMemEval and other community benchmarks aren’t covered head-to-head.
