Memorizing Transformers (Wu et al., 2022)

researchSource PDF: wu-2022-memorizing-transformers.pdf
Part of
Syllabus — Memory for AI agents

Summary

Memorizing Transformers (Wu, Rabe, Hutchins, Szegedy — ICLR 2022) is the canonical “outside-channel” baseline in the AI-agent-memory taxonomy. Rather than expand attention or rewrite weights, a single Transformer layer is augmented with a non-differentiable approximate kNN lookup over a cache of past (key, value) pairs. At inference, the layer attends to (1) the standard local context and (2) the top-k nearest keys retrieved from the external memory, and gates between them with a learned scalar. The model is trained end-to-end (despite the non-differentiable lookup) and shown to extend useful context across long-document tasks — code, mathematical papers (arXiv), formal proofs (Isabelle), books (PG-19), and C4 — with memory banks scaled up to 262k tokens. Perplexity improves monotonically with memory size, and the largest gains are on highly structured/repetitive domains (math, code, formal theorems) where rare-name recall matters.

Key points

Memory architecture

  • A single layer of the Transformer is modified (the authors place it deep in the stack, ~9th of 12). All other layers are vanilla self-attention.
  • The memory is a (key, value) cache of previously-seen positions, not a parametric store. Keys and values are the same tensors used by attention at the memory layer.
  • A batch of past segments is appended to the memory bank as it is processed. At test time, the bank can be arbitrarily larger than the training-time bank — performance keeps improving up to 262k entries.
  • Retrieval is approximate kNN (ScaNN-style) over the keys; the top-k retrieved entries form an external attention context for the current query.

Gated attention

For each query, the layer computes: - local self-attention output $o_\text{local}$ (standard windowed attention), - external attention output $o_\text{ext}$ (attention over the top-k retrieved (k,v) pairs), - a learned per-head gate $g \in [0,1]$, - final output $o = (1-g) o_\text{local} + g o_\text{ext}$.

The gate gives heads the freedom to be “memory heads” or “local heads”; in practice some heads strongly prefer the external pathway.

Training & generalization

  • Trained on long-document corpora at fixed segment length, with the memory growing across segments within a document but cleared across documents.
  • Critically, the memory size at inference can far exceed the size used at training: a model trained with a 8k memory still benefits from extending to 65k or 262k at eval. This is the property δ-mem and Titans later try to recover with state, not retrieval.
  • Reported perplexity wins across PG-19, arXiv-Math, GitHub, C4, Isabelle, with the bigger relative gains on the structured domains.

Why it became the outside-channel baseline

  • Conceptually simple — one extra attention pathway, one gate, one cache.
  • Demonstrated that a frozen-memory, kNN-retrieved store could substantially help next-token prediction without any architectural surgery to the rest of the model.
  • Influenced LongMem (Wang 2023), Landmark Attention, and the whole “retrieval-augmented attention” line.

Limitations (and why the field moved on)

  • Non-differentiable retrieval: gradients don’t flow into the memory; the cache is a frozen snapshot of past KV. Drift / mismatch when training distribution and serving distribution differ.
  • Latency: kNN over hundreds of thousands of entries adds non-trivial inference cost; gate scalar doesn’t fix this.
  • Memory grows linearly with history — not a true compression of past information, just a bigger lookup table.
  • No explicit forgetting / consolidation — every past KV stays in the bank until manually flushed.

δ-mem (Lei 2026) and Titans (Behrouz 2024) explicitly position themselves against this baseline: same goal (use history beyond context), but with a compressed state and a backbone-aligned steering pathway instead of a kNN cache.
