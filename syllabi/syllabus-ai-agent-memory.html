Syllabus — Memory for AI agents

What this covers

Long-horizon LLM agents (Letta, Codex, Claude Code, autonomous research agents) need to accumulate, update, and reuse historical information across sessions — and “just make the context window bigger” doesn’t solve it. Million-token contexts still suffer from context rot (Hong et al. 2025), and quadratic attention is expensive. This syllabus traces the design space: from the earliest external kNN memory (Memorizing Transformers, 2022) through the textual-memory production line (MemGPT, Generative Agents, Mem0) to the latest online-state line (Titans, δ-mem). After reading you should understand the four-paradigm taxonomy of memory mechanisms, see why each paradigm exists, know the delta-rule kernel that powers the modern online-state methods, and have a calibrated sense of which approach fits which problem.

Prerequisites

  • Memory mechanisms for AI agents — a 4-paradigm taxonomy — the 4-paradigm frame (textual / outside-channel / parametric / online-state) used throughout the syllabus. Read first; everything else hangs off it.
  • RAG vs persistent agent memory — why “just do RAG over conversation history” is the wrong default, and how agent memory differs from RAG.
  • Associative memory and the delta rule — math kernel for the modern online-state line: Hopfield-style associative recall, the delta rule as SGD on $\tfrac{1}{2}\|Sk - v\|^2$, gating for forgetting.

Reading order

  1. Memorizing Transformers (Wu et al., 2022) — the canonical outside-channel baseline. A non-differentiable kNN cache attached to one Transformer layer, with a learned gate to mix local and external attention. Demonstrates that external memory works in principle and that gains scale with cache size. Read for the cleanest exposition of the OMM paradigm before anything else replaces it.

  2. Memgpt (Packer et al., 2023) — the canonical textual memory paper and the OS framing. Tier hierarchy (main context / recall / archival), LLM-emitted function calls for self-managed paging, interrupt-driven control loop. Read for the vocabulary (working context, archival storage) that every later TMM paper inherits.

  3. Generative Agents (Park et al., 2023) — the same year as MemGPT, the same paradigm (textual), but introduces the memory stream + reflection loop that becomes the second pillar of modern TMM. Read for: importance scoring at write time, the recency × importance × relevance retrieval, and the empirical case that reflection is what makes memory actually useful.

  4. Titans (Behrouz et al., 2024) — the inflection point. Recasts memory as online optimization: a neural memory module updated at test time by gradient descent on a surprise loss, with momentum and forgetting. Read for the conceptual leap from “retrieve from a static cache” to “learn at inference time” — and for the Memory-as-Context / Memory-as-Gate / Memory-as-Layer integration patterns.

  5. Mem0 (Chhikara et al., 2025) — the production-shaped TMM. The contribution is operational, not algorithmic: ADD / UPDATE / DELETE / NOOP as the right primitives, with LLM-as-judge for each turn. Read for the LOCOMO benchmark (now the standard) and for how textual memory looks when it grows up.

  6. Delta MEM (Lei et al., 2026) — finally, the synthesis. Compact $8\times 8$ online-state memory, updated by gated delta-rule, read as a low-rank correction into a frozen full-attention backbone. Read for the unified 4-paradigm taxonomy, the parameter-efficiency claim (0.12% trainable), and the striking no-context recovery result (8×8 state lifts HotpotQA EM from 0.08 → 6.48).

Implementations

  • AI agent memory — implementation landscape — Letta (open-source MemGPT successor, ~22k stars, Apache 2.0) and Mem0 (open-source production memory layer, ~54k stars, Apache 2.0). Research code for Titans/δ-mem exists but is not production-ready as of May 2026.

Open questions

  • Latent memory vs textual memory head-to-head. δ-mem (Lei 2026) doesn’t benchmark against Mem0 (Chhikara 2025); the comparison everyone wants is missing. The two paradigms may also be complementary — no library combines them.
  • Forgetting curves. Every method handles forgetting differently: MemGPT FIFOs the working context, Generative Agents uses exponential recency decay, Mem0 forgets only on contradiction, Titans/δ-mem have learned per-dimension forget gates. No empirical comparison of which forgetting policy serves which workload.
  • Memory composition across paradigms. Should a production agent in 2026 run textual memory for user-facing facts and online-state memory for low-level steering? Nobody has built this; the design space is wide open.
  • Online memory in agent loops, not just sequences. Titans/δ-mem are evaluated on long sequences, not on multi-tool agent trajectories with branching, tool calls, and back-and-forth. Whether the online-state kernel survives that setting is unknown.
  • The “memory of what was previously remembered” problem. Reflection (Generative Agents) and consolidation are LLM calls over memories. As memory scales, the reflection prompt itself overflows context. No principled solution yet.
  • Standard benchmarks. LOCOMO and MemoryAgentBench are the closest to a standard, but coverage is uneven (LOCOMO is synthetic conversational; MemoryAgentBench covers retrieval / TTL / selective forgetting / long-range understanding). A unified leaderboard would clarify the field.
