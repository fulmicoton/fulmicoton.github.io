AI agent memory — implementation landscape

The two dominant open-source production-shaped agent-memory libraries as of mid-2026, plus the research code for the latent-memory line.

Letta (formerly MemGPT)

  • Repo: https://github.com/letta-ai/letta
  • Stars: ~22k as of May 2026
  • License: Apache 2.0
  • Language: Python; backend service + SDK
  • Last release: v0.16.7, March 2026 (active)

What it implements: - The full Memgpt (Packer et al., 2023) architecture: main context (with subsections), recall storage (vector-indexed conversation history), archival storage (long-term documents). - LLM-emitted function calls for memory management: core_memory_append, core_memory_replace, archival_memory_insert, archival_memory_search, conversation_search. - Multi-step agent loop with request_heartbeat to chain function calls before responding. - Web UI for inspecting agent state, memory contents, function call traces. - Multi-tenant deployment via the Letta server.

What it does not implement: - The latent-memory line (Memorizing Transformers, Titans, δ-mem) — Letta is squarely in the textual-memory paradigm. - Automatic reflection (à la Generative Agents) — there are recipes for it but it’s not built in. - Forgetting curves — eviction is FIFO from working context to recall.

Notable in the 2026 timeline: - 2024-09: MemGPT rebranded to Letta; the company maintains the project. - 2026-03: v0.16.7 raised the default working context from 32k to 128k and overhauled compaction.

Mem0

  • Repo: https://github.com/mem0ai/mem0
  • Stars: ~54k as of May 2026 (much wider adoption than Letta)
  • License: Apache 2.0
  • Language: Python; SDK + hosted API
  • Maintenance: Active commits as of 2026-05.

What it implements: - The Mem0 (Chhikara et al., 2025) pipeline: per-turn fact extraction → search → ADD/UPDATE/DELETE/NOOP decision → vector-store apply. - Pluggable vector stores (Qdrant, Chroma, pgvector, Pinecone, etc.). - Mem0g graph variant for relational memory (uses Neo4j or similar). - Chrome extension + MCP server for stuffing Mem0 into existing chat apps (ChatGPT, Claude, Perplexity, Grok). - Hosted SaaS at mem0.ai with persistence and team features.

What it does not implement: - The latent / online-state memory line. - Self-managed function-call style (the agent doesn’t “choose” to remember; extraction is automatic per turn). - Bounded context tier hierarchy à la MemGPT — memory is one flat vector store (or a graph).

Comparison vs Letta: - Letta: agent is the system. The LLM autonomously manages tiers via function calls. Good fit if you’re building a stateful assistant from scratch. - Mem0: drop-in memory layer for existing agents. The agent isn’t aware of Mem0 — it just calls mem0.add(...) and mem0.search(...). Good fit if you have an existing chat app and want to bolt on persistence.

Research code for the latent-memory line

Titans

  • Repo: not officially released by Google as of May 2026 (paper has no public code link).
  • Multiple community PyTorch implementations on GitHub (lucidrains/titans-pytorch, etc.) of varying fidelity.

δ-mem

  • Repos (declared in the paper): https://github.com/Declare-lab and https://github.com/MindLab-Research
  • Paper dated 2026-05-12; code was being released in the same window.
  • Status as of 2026-05-17 (paper-release week): code exists but is not yet battle-tested in production; fidelity to the paper varies across forks.

These are research artifacts, not production libraries. For deploying online-state memory in a real agent today, the practical answer is: there isn’t one yet. The latent-memory line is on the cusp.

What’s missing in the landscape

  • No mature library that combines textual memory with online-state memory. The two paradigms are complementary (textual for explainable facts, online-state for low-level steering) but no framework integrates both.
  • No standard benchmark suite that all libraries report against. Mem0 reports LOCOMO; δ-mem reports LoCoMo + MemoryAgentBench + HotpotQA. Letta reports its own internal eval.
  • Letta and Mem0 don’t directly bench against each other in the literature. Anecdotally Mem0 has wider adoption; Letta has the richer agent-state model.
