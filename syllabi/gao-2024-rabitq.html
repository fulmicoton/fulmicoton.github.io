Syllabus — TurboQuant and the rotation-based quantization line

Summary

RaBitQ (Gao & Long, SIGMOD 2024) is a randomized 1-bit-per-dimension quantization method for high-dimensional Euclidean vectors used in approximate nearest-neighbor search. It centers and unit-normalizes the database vectors, then quantizes each vector to a vertex of a randomly rotated $\{\pm 1/\sqrt{D}\}^D$ hypercube — equivalent to storing the sign of each coordinate after a random orthogonal rotation. The paper proves an asymptotically tight $O(1/\sqrt{D})$ error bound on the inner-product estimator (which Alon & Klartag 2017 show is the best possible for $D$-bit codes), giving RaBitQ a theoretical guarantee that PQ-family methods lack. Empirically it beats product quantization at half the code length on standard ANN benchmarks; the SIMD-friendly bitwise distance estimator is ~3× faster than the original PQ table-lookup. It is the direct conceptual predecessor — and now competitor — of TurboQuant’s first 1-bit stage.

Key points

Algorithm

  1. Center & normalize: subtract dataset centroid, scale each $x$ to unit norm $\bar{o}$.
  2. Random rotation: pick a random orthogonal $P$; the quantization codebook is $\{P v : v \in \{\pm 1/\sqrt{D}\}^D\}$.
  3. Code assignment: code of $\bar{o}$ is the sign pattern $\text{sign}(P^{-1}\bar{o})$ packed as $D$ bits.
  4. Query: rotate the query $\bar{q}$, scalar-quantize to a few bits, compute the inner product of the two rotated vectors using bitwise / SIMD ops, then re-rank with raw vectors.

Unbiased estimator

The core estimator $\langle \tilde{o}, \bar{q}\rangle / \langle \tilde{o}, \bar{o}\rangle$ is unbiased for $\langle \bar{o}, \bar{q}\rangle$, where $\tilde{o}$ is the quantized vector. The denominator $\langle \tilde{o}, \bar{o}\rangle$ is precomputed and stored alongside the code — this is RaBitQ’s one per-vector scalar overhead (TurboQuant later replaces this with the precomputed Lloyd-Max constants of the Beta distribution).

Asymptotic optimality

The $O(1/\sqrt{D})$ error bound matches the Alon–Klartag 2017 lower bound for any $D$-bit binary code on the unit sphere. So in the 1-bit-per-dim regime RaBitQ is rate-distortion optimal up to constants — which is precisely why TurboQuant’s 1-bit residual stage uses the same structural recipe.

Empirical results vs PQ

  • Recovers higher recall than PQ at half the code length on standard ANN datasets (MSong, GIST, DEEP1M, SIFT1M).
  • On MSong, PQ shows >50% relative distance error and <60% recall even with re-ranking — RaBitQ’s theoretical bound prevents this kind of dataset-specific blow-up.
  • Single-vector implementation: bitwise XOR + popcount → ~3× faster than PQ’s table lookups.
  • Batched implementation: comparable to fast-scan PQ variants while keeping the error bound.

Why exactly 1 bit per dimension

The construction is tied to sign-bit quantization on the sphere — extending to multi-bit RaBitQ requires either repeating the construction independently (multi-bit, multi-rotation) or moving to a scalar quantizer per coordinate. The latter is essentially what TurboQuant does, so TurboQuant subsumes RaBitQ as the 1-bit special case while generalizing to any $b$.

How RaBitQ differs structurally from TurboQuant

RaBitQ TurboQuant
Rate 1 bit/dim fixed any $b$ bits/coord
Per-vector overhead $\langle \tilde{o}, \bar{o}\rangle$ scalar $\|r\|_2$ scalar (in residual stage)
Bias Unbiased by ratio estimator MSE stage biased, fixed by QJL residual
Application targeted ANN search KV cache + ANN
