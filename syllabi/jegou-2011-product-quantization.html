Product Quantization (Jegou et al., 2011)

researchSource PDF: jegou-2011-product-quantization.pdf

Syllabus — TurboQuant and the rotation-based quantization line

Summary

Product Quantization (Jégou, Douze, Schmid — IEEE TPAMI 2011) is the canonical method for compressing high-dimensional vectors for approximate nearest neighbor search. It splits each $d$-dim vector into $m$ sub-vectors of dimension $d/m$, runs k-means independently on each subspace to learn a codebook of $k$ centroids, and represents each input vector by the concatenation of the $m$ centroid indices — $m \log_2 k$ bits per vector. Distance from a query to any database vector is computed by precomputing a $m \times k$ lookup table of squared distances between query sub-vectors and centroids, then summing $m$ lookups per database vector. The 2011 paper introduced both symmetric (quantize both query and DB) and asymmetric (quantize only DB) estimators, paired PQ with an inverted file index (IVFADC) for billion-scale search, and held the empirical state-of-the-art on SIFT1B for nearly a decade. PQ is the baseline every modern quantization paper — RaBitQ, TurboQuant, OPQ, ScaNN — measures itself against.

Key points

Construction

  • Split $x \in \mathbb{R}^d$ into $m$ sub-vectors $x = [x^1, \ldots, x^m]$, each of dim $d/m$.
  • Train independent k-means codebooks $C^1, \ldots, C^m$, each with $k = 256$ (so each index fits in 1 byte).
  • Quantization code: $q(x) = (q^1(x^1), \ldots, q^m(x^m))$ — $m$ bytes per vector.
  • Typical config: $d=128$ SIFT, $m=8$ → 64 bits per vector for SIFT1B.

Asymmetric distance computation (ADC)

Given query $y$: 1. Precompute table $T[i][j] = \|y^i - c_j^i\|^2$ for each sub-space $i$ and centroid $j$. 2. For each database vector with code $(j_1, \ldots, j_m)$: $\hat{d}(y, x)^2 = \sum_i T[i][j_i]$.

This turns an O(d) distance per DB vector into O(m) lookups. The asymmetry (query unquantized, DB quantized) keeps more accuracy than symmetric SDC where both are quantized.

IVFADC: inverted index + residual PQ

For billion-scale search, the paper adds a coarse quantizer (a single k-means with ~$10^3$–$10^4$ cells) that partitions the dataset, then runs PQ on the residuals $(x - \text{coarse}(x))$ inside each cell. Search only scans the cells nearest to the query — sublinear in dataset size. This is the architecture that powered Facebook’s faiss for years.

Empirical results (2011)

Dataset Bits/vec Recall@100
SIFT1M 64 ~95%
SIFT1B 64 ~80%
GIST1M 64 ~90%

Search speed: ~10 ms per query on SIFT1B with IVFADC on a single 2010-era CPU.

Limitations later attacked by RaBitQ / TurboQuant

  • No theoretical error bound. PQ relies on k-means converging to “good” centroids; on adversarial distributions it can fail catastrophically (50% error on MSong — see RaBitQ (Gao et al., 2024)).
  • Expensive training: k-means on each sub-space, repeated for residual stages — minutes to hours on 1M-vec datasets, days on 1B.
  • Not online: codebooks are tied to the dataset distribution; new data drifts away from learned centroids.
  • Subspace independence assumption: improved by OPQ (Ge et al. 2013) and later by rotation-then-quantize approaches like RaBitQ.
